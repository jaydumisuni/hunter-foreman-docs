# Screenshot Capture List

Capture these after the connected demo is running.

## Required Screenshots

1. **Main landing / ROSE intake**
   - Show Hunter Foreman title and request form.

2. **Loaded demo scenario**
   - Show event booking request loaded in the form.

3. **Foreman dashboard task**
   - Show workflow, confidence, owner, status, and escalation result.

4. **App bridge status**
   - Show connected mode and last dispatch status.

5. **Receiver app task**
   - Show received task in the external app.

6. **Receiver timeline**
   - Show ROSE → Foreman → AppBridge timeline.

7. **Docker / terminal running**
   - Optional, useful for proving judge-friendly setup.
