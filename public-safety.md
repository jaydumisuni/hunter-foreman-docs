# Public Safety Boundary

Hunter Foreman is part of a larger Hunter ecosystem, but this hackathon submission is intentionally public-safe.

## Included Publicly

- ROSE intake demo
- Foreman routing examples
- task creation
- workflow labels
- dashboard demo
- notification previews
- app bridge contract
- receiver app
- Docker and CI setup
- submission documentation

## Not Included Publicly

- private Hunter internals
- private prompts
- production credentials
- real client data
- payment credentials
- private business workflows
- device repair or recovery modules
- unlock or bypass logic
- proprietary THETECHGUY operational secrets

## Why This Boundary Exists

The public repo must let judges and contributors understand the product without exposing sensitive business systems or client data.

The public project demonstrates the operating model. Private production modules can later integrate only after the public demo survives the break/test phase.

## Rule For Future Contributions

If a contribution risks exposing private credentials, client data, sensitive repair/recovery logic, or private Hunter internals, it does not belong in the public repository.
